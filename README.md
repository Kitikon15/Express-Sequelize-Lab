## 1. สิ่งที่ต้องติดตั้งก่อนเริ่มใช้งาน (Prerequisites)
1. **Node.js** (เวอร์ชัน 16 ขึ้นไป)
2. **Docker Desktop** (สำหรับรัน PostgreSQL)
3. **DbGate** (โปรแกรมจัดการฐานข้อมูล)
4. **Postman** (โปรแกรมทดสอบ API)

---

## 2. ขั้นตอนการติดตั้งและเริ่มรันระบบ (Setup & Running)

เปิด Terminal ในโฟลเดอร์โปรเจกต์ (`d:\BackEnd-Lab\Express-Sequelize-Lab`) แล้วรันคำสั่งดังนี้:

### ขั้นตอนที่ 2.1: ติดตั้ง Dependencies
```bash
npm install
```

### ขั้นตอนที่ 2.2: รัน PostgreSQL Database ด้วย Docker
รันคำสั่งเพื่อสร้างและเปิดใช้ฐานข้อมูลในรูปแบบ Background (Background service):
```bash
docker compose up -d
```
*หมายเหตุ: สามารถตรวจสอบความเรียบร้อยได้โดยรัน `docker ps` จะต้องเห็น Container ชื่อ `express_postgres_container` กำลังทำงานอยู่*

### ขั้นตอนที่ 2.3: สตาร์ทเซิร์ฟเวอร์ (Development Mode)
รันคำสั่งนี้เพื่อให้เซิร์ฟเวอร์เริ่มทำงานผ่าน Nodemon (เมื่อแก้ไขโค้ดจะรีสตาร์ทตัวเองอัตโนมัติ):
```bash
npm run dev
```
เมื่อรันสำเร็จ จะต้องขึ้นข้อความบนหน้าจอ Terminal ดังนี้:
```text
🚀 Server running on http://localhost:5000
Connection to PostgreSQL!
Table synchronized !
```

## 3. การเชื่อมต่อฐานข้อมูลด้วย DbGate

**DbGate** คือโปรแกรมที่ใช้เข้าไปเปิดดูข้อมูลในตาราง แก้ไขข้อมูล หรือตรวจสอบโครงสร้างฐานข้อมูลแบบ GUI (หน้าต่างกราฟิก)

### ขั้นตอนการเชื่อมต่อ:
1. เปิดโปรแกรม **DbGate**
2. คลิกที่ปุ่ม **New Connection** (หรือคลิกขวาที่แถบด้านซ้าย เลือก Add connection)
3. ในช่อง **Database Engine** ให้เลือกเป็น `PostgreSQL`
4. กรอกข้อมูลการเชื่อมต่อดังต่อไปนี้ (อ้างอิงตามไฟล์ `db.js` และ `Docker-compose.yml`):
   * **Server / Host:** `localhost`
   * **Port:** `5433` (ห้ามสับสนกับพอร์ตมาตรฐาน 5432 เนื่องจากเรา Map พอร์ตภายนอกเป็น 5433)
   * **User:** `dev_user`
   * **Password:** `dev_password`
   * **Database:** `products_db`
5. กดปุ่ม **Test** เพื่อทดสอบการเชื่อมต่อ (ถ้าถูกต้องจะขึ้นตัวอักษรสีเขียวว่า Success)
6. กดปุ่ม **Save** หรือ **Connect**
7. เมื่อเชื่อมต่อแล้ว ที่แถบด้านซ้ายให้กดขยายโฟลเดอร์:
   * `products_db` > `schemas` > `public` > `tables`
   * คุณจะเจอตารางชื่อ **`Products`**
   * ดับเบิ้ลคลิกที่ตาราง `Products` แล้วคลิกแท็บ **Data** เพื่อดูรายการสินค้าทั้งหมดในฐานข้อมูล หรือคลิกแท็บ **Structure** เพื่อดูคอลัมน์ (`id`, `name`, `price`)

## 4. การทดสอบ API ด้วย Postman (ทุกขั้นตอน)

API นี้เปิดพอร์ตไว้ที่ `http://localhost:5000` โดยมีเส้นทาง (Endpoints) หลักๆ ดังนี้:

| วิธีการ (Method) | เส้นทาง (URL) | อธิบาย |
| :--- | :--- | :--- |
| **GET** | `/api/products` | ดึงข้อมูลสินค้าทั้งหมด |
| **GET** | `/api/products/:id` | ดึงข้อมูลสินค้าเฉพาะชิ้นตาม ID |
| **POST** | `/api/products` | เพิ่มสินค้าใหม่เข้าฐานข้อมูล |
| **PUT** | `/api/products/:id` | แก้ไขข้อมูลสินค้าเดิมตาม ID |
| **DELETE** | `/api/products/:id` | ลบสินค้าออกจากฐานข้อมูลตาม ID |

*นอกจากนี้ยังมีระบบอำนวยความสะดวก: หากเปิดเว็บบราวเซอร์ไปที่หน้าแรกสุด `http://localhost:5000/` ระบบจะส่งผู้ใช้ไปที่ `/api/products` โดยอัตโนมัติ*

---

### ขั้นตอนการเรียกทดสอบใน Postman ทีละหน้าจอ:

#### 1. การเพิ่มสินค้าใหม่ (POST - Create)
* **Method:** เลือกเป็น **`POST`** จาก Dropdown ด้านหน้า
* **URL:** กรอก `http://localhost:5000/api/products`
* **การส่งข้อมูล (Body):**
  1. คลิกแท็บ **`Body`** (อยู่ใต้ช่อง URL)
  2. ติ๊กเลือกประเภทเป็น **`raw`**
  3. ด้านขวาสุดของแถวตัวเลือก Body จะมีคำว่า `Text` ให้คลิกเปลี่ยนเป็น **`JSON`**
  4. เขียนข้อมูลสินค้าที่ต้องการเพิ่มลงในกล่องข้อความดังนี้:
     ```json
     {
       "name": "Sony Headphones",
       "price": 299.99
     }
     ```
  5. กดปุ่ม **`Send`** สีน้ำเงินขวาบน
  6. **ผลลัพธ์ตอบกลับ:** ด้านล่างจะแสดงข้อมูลสินค้าที่มี ID ถูกสร้างขึ้นใหม่ในฐานข้อมูล พร้อมสถานะ `201 Created`

#### 2. การดึงข้อมูลสินค้าทั้งหมด (GET - Read All)
* **Method:** เลือกเป็น **`GET`**
* **URL:** กรอก `http://localhost:5000/api/products`
* **การส่งข้อมูล:** ไม่ต้องตั้งค่า Body
* **ปุ่มกด:** กดปุ่ม **`Send`**
* **ผลลัพธ์ตอบกลับ:** จะได้รับรายการสินค้าทั้งหมดที่อยู่ในรูปแบบ Array ของ JSON พร้อมสถานะ `200 OK`

#### 3. การค้นหาสินค้าชิ้นเดียว (GET - Read One)
* **Method:** เลือกเป็น **`GET`**
* **URL:** กรอก `http://localhost:5000/api/products/1` *(เลข 1 คือ id ของสินค้าที่ต้องการค้นหา)*
* **ปุ่มกด:** กดปุ่ม **`Send`**
* **ผลลัพธ์ตอบกลับ:** จะได้รับรายละเอียดของสินค้านั้น หากไม่พบระบบจะแจ้งเตือนด้วยรหัสสถานะ `404 Not Found`

#### 4. การแก้ไขข้อมูลสินค้า (PUT - Update)
* **Method:** เลือกเป็น **`PUT`**
* **URL:** กรอก `http://localhost:5000/api/products/1` *(เลข 1 คือ id ของสินค้าที่จะทำการแก้ไข)*
* **การส่งข้อมูล (Body):**
  1. คลิกแท็บ **`Body`** -> เลือก **`raw`** -> เปลี่ยนเป็น **`JSON`**
  2. ใส่ข้อมูลส่วนที่ต้องการแก้ไข (สามารถอัปเดตเฉพาะชื่อ หรือเฉพาะราคา หรืออัปเดตทั้งหมดได้):
     ```json
     {
       "name": "Sony Headphones Wireless",
       "price": 349.99
     }
     ```
  3. กดปุ่ม **`Send`**
  4. **ผลลัพธ์ตอบกลับ:** ข้อมูลสินค้าที่ถูกแก้ไขแล้วจะแสดงผลกลับมา พร้อมสถานะ `200 OK`

#### 5. การลบสินค้าออก (DELETE - Delete)
* **Method:** เลือกเป็น **`DELETE`**
* **URL:** กรอก `http://localhost:5000/api/products/1` *(เลข 1 คือ id ของสินค้าที่จะลบ)*
* **ปุ่มกด:** กดปุ่ม **`Send`**
* **ผลลัพธ์ตอบกลับ:** ด้านล่างจะขึ้นข้อความระบุว่า `{"message": "Product deleted successfully"}` พร้อมสถานะ `200 OK`
